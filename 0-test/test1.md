5. 块级元素和内联元素分别有哪些？有什么区别？
块级元素：
  代表页面上的块，通常会在页面中单独占据一行或者一块空间。
  常见的块级元素包括 <div>、<p>、<h1> 到 <h6>、<ul>、<li>、<table> 等。
  块级元素可以包含内联元素和其他块级元素。
  设置块级元素的宽度、高度、边距和填充等样式属性是有效的。
内联元素：
  代表文档中的行内内容片段，它们不会打断当前行的流程，与相邻的内联元素在同一行上显示。
  常见的内联元素如 <span>、<a>、<strong>、<em>、<img> 等。
  内联元素通常只能包含文本或者其他内联元素。
  设置内联元素的宽度和高度等尺寸属性通常不会生效，但可以设置水平方向的边距和填充。

2. 列出你知道的CSS伪类。简单描述下各自的作用。
  1、:hover： 当鼠标悬停在元素上时应用样式。常用于创建鼠标悬停效果，比如链接的颜色变化等。
  2、:active： 当元素被激活或点击时应用样式。通常用于按钮或链接，表示用户正在与之交互。

  4、:visited： 选择用户已访问过的链接，允许为已访问链接定义不同的样式。
  5、:nth-child()： 选择作为父元素中特定位置的子元素的元素。

3. div和span的差别？
  <div>：
  
  是块级元素，它在页面中通常以块的形式显示，独占一行或一块空间。
  用于组织和包装一组相关的内容，常用于创建页面布局和分组大块内容。
  通常用于包裹一组元素，如导航菜单、页脚、侧边栏等，并且可以应用于CSS来设置样式。
  <span>：
  
  是内联元素，它在行内显示，不独占一行，与相邻元素在同一行上显示。
  用于包裹文本或者行内元素的小片段，并且通常用于为这些片段设置样式或添加特定的行为。
  经常用于对文本的特定部分应用样式，或者包裹行内的元素。

4. 如何让块级元素居中？如何让内联元素居中？
  块级元素居中：
  使用 Flex容器，align-items: center; 实现垂直居中。
  以及justify-content: center;实现水平居中。

  内联元素居中：使用 text-align: center;将包含内联元素的父元素设置为 text-align: center; 来实现内联元素的水平居中。内联元素默认不支持垂直居中。但可以通过设置父元素的高度、行高以及 line-height 等属性来模拟实现一定程度的垂直居中效果。
5. 如何实现垂直居中？

  块级元素居中：使用 Flex容器，align-items: center; 实现垂直居中。
  内联元素居中：内联元素默认不支持垂直居中。但可以通过设置父元素的高度、行高以及 line-height 等属性来模拟实现一定程度的垂直居中效果。

2. (可选）什么是数据类型？
数据类型是编程中用于定义数据的性质和操作的分类或种类。在计算机编程中，数据类型定义了变量或对象可以存储的数据种类，以及对该数据可以执行的操作类型。不同的编程语言具有不同的数据类型。

3. Javascript有哪些数据类型？
  基本类型（原始类型）：
    字符串（String）： 用于表示文本数据，使用单引号 ' 或双引号 " 括起来的字符序列。
    数字（Number）： 用于表示数值，包括整数和浮点数。
    布尔值（Boolean）： 用于表示逻辑值，只有两个值，true 和 false。
    空（Null）： 表示一个空值或空对象引用。
    未定义（Undefined）： 表示未定义或未初始化的变量值。
    符号（Symbol）（ES6新增）： 表示唯一的、不可变的值。
  复合类型：
    对象（Object）： 用于存储多个键值对，或者复杂数据结构。
    数组（Array）： 一种特殊的对象，用于存储按序排列的数据集合。
    函数（Function）： 一种可执行的代码块，可以被调用和重复使用。

4. 什么事函数？如何调用函数？
    函数是一段封装了特定任务或操作的可重复使用的代码块。它们允许您将代码逻辑组织成一个单独的单元，当需要执行特定任务时，可以通过函数调用来执行这些代码。
    如：
    function functionName(parameter1, parameter2, ...) {
      return result; // 可选，返回函数执行的结果
    }
    调用：
    functionName(argument1, argument2, ...);

5. (可选）NaN是什么？属于上面哪种数据类型？如何检查某个变量是否是NaN？
  1.NaN 代表 "Not a Number"，是一个特殊的数值，表示一个无效的或不可表示的数值。当某些数学操作无法返回有效的数值时，就会得到 NaN。例如，0 除以 0、对非数值字符串进行数学运算等都会得到 NaN。

  2.要检查某个变量是否是 NaN，可以使用 isNaN(value) ，并在判断之前会尝试将参数转换为数字类型。这个函数接受一个参数，并返回一个布尔值，指示参数是否为 NaN。如果 value 是 NaN，isNaN(value) 返回 true，否则返回 false。
  严格模式下建议使用Number.isNaN()，只有当参数是严格的 NaN 时才会返回 true。

6. (可选) 什么是变量提升？
  变量提升指的是在代码执行前阶段，JavaScript 引擎会将变量声明移动（提升）到其作用域的顶部，但只是声明被提升了，赋值不会提升。即无论在何处声明变量，它们都会在执行代码之前被“提升”到当前作用域的顶部。

7. (可选）事件冒泡和捕获有什么区别？
  事件冒泡（Event Bubbling）：
  事件开始时，首先发生在最具体的元素（最内部的元素），然后逐级向上传播至最不具体的元素（最外部的元素）。
  比如，当某个子元素上的事件被触发时，事件将首先在该子元素上被处理，然后沿着 DOM 树向上传播至父元素，一直传播至最外部的祖先元素。
  大多数情况下，事件默认采用冒泡方式进行传播。

  事件捕获（Event Capturing）：
  事件开始时，从最不具体的元素（最外部的元素）开始，然后逐级向下传播至最具体的元素（最内部的元素）。
  在捕获阶段，事件会首先在最外部的元素上被触发，然后沿着 DOM 树向下传播至最具体的目标元素。
  在标准的事件流中，捕获阶段很少被利用，因为大多数情况下默认使用的是冒泡阶段。

  区别：
  方向： 冒泡是由最具体的元素向最不具体的元素传播，而捕获是从最不具体的元素向最具体的元素传播。
  顺序： 冒泡是自底向上的传播，而捕获是自顶向下的传播。
